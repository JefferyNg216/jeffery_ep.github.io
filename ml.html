                        Completing the activities provides a clear understanding of how to choose between linear and polynomial regression, which are:-

                      • Correlation Coefficient:
                          This statistic is key in choosing the regression type. A correlation coefficient near 1 or -1 indicates a strong linear relationship, favoring linear regression. A coefficient near 0 suggests a weak linear relationship, possibly necessitating polynomial regression for curvilinear data patterns.

                      • Visual Inspection:
                          Scatter Plots - Plotting data can visually expose the relationship type. A linear trend suggests linear regression, while a curved pattern may better suit polynomial regression to capture relationship nuances.

                        • Therefore, the decision between linear and polynomial regression should be based on statistical correlation measures and visual data analysis. This dual approach ensures a robust model selection process.

                        First, I've created the figure below (Figure 1) which illustrates the operation of a Multi-layer Perceptron (MLP) model to assist me in understanding the Neural Networks, tailored for ensemble level learning using methods such as bagging or boosting. The network's architecture consists of multiple layers:

                      • Input Layer: This layer accepts the initial data inputs, which in this example could be various features of a dataset or specific questions. Each neuron here symbolizes an individual feature or a unique aspect of the input.

                      • Hidden Layers: These layers process the inputs from the previous layer through weighted connections. Each neuron applies a non-linear transformation to its inputs, which are then forwarded to the subsequent layer. The complexity and depth of these layers enhance the model's ability to learn from complex datasets by capturing intricate non-linear relationships.

                      • Output Layer: This final layer generates the prediction or classification results based on the processed inputs from the preceding layers. In this scenario, it could represent an answer or decision derived from the processed inputs.
                        • (Jorgecardete, 2024)


                        Figure 1. Neural Network Structure

                        After having established a basic understanding of neural networks above, Figure 2 delves into the specifics of neuron interactions within a neural network, which include:

                      • Weight Initialization: Each connection from one neuron to another has a weight that is initially set.
                      • Activation Function: This function transforms the weighted input from the neuron into an output that is sent to the next layer.
                      • Forward Pass: The output is passed forward through the network to the output layer.
                      • Loss Calculation: At the output layer, the network calculates the error (loss) between the predicted output and the actual output.
                      • Backpropagation: If the output isn’t accurate, the error is sent back through the network. This backpass adjusts the weights using a method called gradient descent to minimize the error.
                      • Weight Update: Based on the error, the weights are updated to produce a more accurate output in the next forward pass.
                        • (Geeksforgeeks, 2024)

                        The process iterates, continually adjusting weights to increase accuracy, and then moves to the next neuron, improving the network's overall performance.

                        Figure 2. Neurons interactions

                        Finally, figure 3 outlines the training process for a machine learning model, specifically a Convolutional Neural Network (CNN). The summary of the steps involved:

                      • Start Training: The process begins by dividing the data into a training set and a validation set. The training set is used to train the model, while the validation set is used to evaluate the model's performance to ensure it generalizes well to new data.
                      • Validation: After training on the training set, the model makes predictions which are then validated against the validation set. This step checks how well the model's predictions match the actual data.
                      • Early Stopping: This mechanism is used to prevent the model from overfitting. The process involves several checks:
                        • Calculate Validation Loss: The loss (error) on the validation set is calculated to gauge the accuracy of the model.
                        • Check for Maximum Epochs: The model checks if the maximum number of training cycles (epochs) has been reached. If not, it assesses whether there is still room for improvement in reducing validation loss.
                        • Room to Improve: If the model can still improve (validation loss can decrease), training continues. Otherwise, the model checks if it has reached the patience limit, which is the number of epochs to wait before stopping when no improvement is seen.
                      • Model Evaluation: Once the training stops—either from reaching the maximum epochs or if there is no improvement even after additional epochs—the model is finally evaluated to see how well it performs overall.
                        • (Dutta S., 2024)

                        Description and Analysis
                        In the learning summary outlined above, the necessary techniques and best practices for the machine learning model are detailed for our professional development. Nevertheless, several challenges and considerations were identified during the assessment implementation.

                      • Applicability and Challenges of data
                          The implementation of the Unit 6 assessment, which involved conducting an Exploratory Data Analysis (EDA) to address specific business inquiries, highlighted the importance of selecting the appropriate data. Decisions about which data to utilize should be primarily driven by the business objectives. Additionally, it is crucial to consider whether the goals are extensive and if the dataset contains enough information to identify relevant issues (Thilakarathne H., 2022). Effective communication with stakeholders and possessing relevant domain knowledge are essential requirements for this process.

                      • Furthermore, several additional challenges were identified, such as:
                        • Data Lack: During the analysis stage, a deficiency in data can occur due to inadequate consideration in the initial stages.
                        • Dark Data: While storing as much data as possible might seem like a solution to the data lack issue, it can lead to problems with dark data, which may significantly increase computational and maintenance resources (IBM, N.D).
                        • High Dimensionality: Datasets with an excessively high number of features can complicate model training, a phenomenon known as the "curse of dimensionality" (Karanam S., 2021).
                          • Besides these specific issues, there are other common data-related challenges in machine learning, including data quality, insufficient training data, and problems related to overfitting or underfitting (Saxena A., 2024).
                      • Real-World Challenges of Machine Learning
                          • From the implementation of the Unit 11 assessment, which involved using CNN models for image identification, it's clear that this practice has become popular and can be applied to various business areas. These include image classification and face recognition for social media and citizen identification, Optical Character Recognition for document digitization, and image computing and predictive analytics in healthcare data science for risk assessment (Bilyk V., 2020).

                        However, this also brings with it a number of debates concerning machine learning, with major focus areas including:

                      • Legal and Privacy
                          While machine learning seamlessly integrates into our social networks, it frequently employs algorithms tailored to predict user preferences in areas such as shopping, music, and leisure activities. Although these predictive capabilities enhance user experience by personalizing content, they also bring with them substantial privacy concerns. The collection and analysis of personal data to feed these algorithms can lead to potential breaches of user confidentiality and raise ethical and legal questions about data use and consent.
                      • Social and Ethical
                          As AI becomes increasingly integrated into job roles and tasks, such as process automation and chatbots, human interaction decreases. ChatGPT is a prime example of this trend. Consider a situation where you encounter a problem while studying; who is your first point of contact? Personally, I often turn to ChatGPT and frequently search Google for answers, instead of colleagues and tutor. Currently, social interaction is already on the decline due to the pervasive use of mobile devices and the internet. With the rapid advancement of machine learning technology, a further decrease in face-to-face interactions is anticipated. (Whyte T. J., 2019)
                        • Although machine learning presents certain challenges as it evolves, in a rapidly changing world it represents a necessary advancement. People must adapt to and regulate this technology to harness its full potential while mitigating its risks.
